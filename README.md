## Algorithm Practice Log

이 레포는 코딩테스트 문제를 단순히 저장하는 목적이 아니라,  
문제를 어떤 사고 과정으로 접근하고 해결했는지를 정리하기 위한 공간입니다.

문제 풀이 결과뿐만 아니라  
접근 전략, 반복되는 실수, 개선 포인트를 함께 기록하는 것을 목표로 합니다.

---

## Repository Structure

### 📁 problems
풀이한 코딩테스트 문제의 **코드만 보관**합니다.  
문제 번호나 출처 기준으로 관리하며,  
풀이 과정에 대한 상세 설명은 별도 문서에서 다룹니다.

---

### 📁 patterns
문제 풀이 과정에서 **반복적으로 사용되는 사고 방식과 접근 전략**을 정리합니다.  
특정 문제 하나가 아니라, 여러 문제에 공통으로 적용되는 내용을 중심으로 작성합니다.

#### ▸ 문제 유형별 접근
- [탐색 (BFS / DFS)] 
- [이분 탐색]  
- [동적 계획법 (DP)]
- [구현 문제 접근]

#### ▸ 사고 패턴 & 방법론
- [문제 풀이 접근 전략](patterns/problem-solving-strategy.md)  
- [시간 복잡도 판단 기준](patterns/time-complexity.md)

#### ▸ 실수 기록 & 체크리스트
- [자주 하는 실수 정리](patterns/common-mistakes.md)  
- [문제 풀이 전 체크리스트](patterns/pre-solve-checklist.md)
- [엣지 케이스 정리](patterns/edge-case.md) 

---

### 📁 featured
문제 선택 이유, 접근 전략, 시행착오 등을 함께 정리한  
**대표 문제 모음**입니다.

면접에서 설명 가능한 문제를 중심으로 선별하며,  
`patterns`에 정리된 사고 방식이 실제 문제에서 어떻게 적용되었는지를 연결해 설명합니다.

※ 새로운 패턴 문서는 필요할 때마다 추가하며, README에 링크를 함께 갱신합니다.
